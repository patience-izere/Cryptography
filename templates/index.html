Polyalphabetic Cipher (Vigenère)

How It Works:

The Vigenère cipher is a polyalphabetic substitution cipher that uses a keyword to determine shifts. Unlike the Caesar cipher (which shifts all letters by the same amount), Vigenère shifts each letter by different amounts based on corresponding keyword letters. This makes it much stronger and harder to break.

Encryption Formula:

Ci = (Pi + Ki mod |K|) mod 26

Where: Ci = ciphertext character, Pi = plaintext character, K = keyword (repeated)

Visual Example:

Plaintext: H E L L O
(7) (4) (11) (11) (14) ← positions
│ │ │ │ │
Keyword: S E C R E
(18) (4) (2) (17) (4) ← shifts
│ │ │ │ │
Add & MOD: (7+18) (4+4) (11+2) (11+17) (14+4) = 25 mod 26
│ │ │ │ │
Ciphertext: Z I N I S ← result

Key Concepts:

  • Keyword repeats: If plaintext is longer than keyword, keyword cycles through
  • 26 alphabets: Each keyword letter creates a different shifting pattern
  • Letter positions: A=0, B=1, C=2, ... Z=25
  • Non-letters: Spaces and punctuation are preserved as-is

Security Note:

⚠️ While stronger than Caesar cipher, Vigenère can still be broken through frequency analysis if the keyword is short or repeated. Modern encryption is much more secure.

El Gamal Encryption (Asymmetric)

How It Works:

El Gamal is an asymmetric encryption algorithm based on the discrete logarithm problem. Unlike Vigenère (which uses one shared key), El Gamal uses a public-private key pair. Anyone can encrypt with the public key, but only the private key holder can decrypt. This is like a mailbox: anyone can put mail in (encrypt), but only you have the key to open it (decrypt).

Three Phases:

1️⃣ Key Generation:
y = gx mod p

Select prime p, generator g
Private key: x (kept secret)
Public key: y (shared with everyone)

2️⃣ Encryption (Anyone can do):
c1 = gk mod p
For each character Mi:
c2[i] = (Mi × yk) mod p
Ciphertext = (c1, [c2[0], c2[1], ...])

Where: k = random number (same for all), Mi = each character (ASCII), p = prime
Each character is encrypted individually using the same k and c₁

3️⃣ Decryption (Only private key holder):
s = c1x mod p
s-1 = s(p-2) mod p
For each c2[i]:
Mi = (c2[i] × s-1) mod p

Uses Fermat's Little Theorem for modular inverse
Each character is decrypted individually from its corresponding c₂ value

Visual Flow:

SENDER RECEIVER
│ │
├─ Sees public key y ────→ │ (Receiver has x, p)
│ │
├─ Chooses random k │
├─ Calculates c₁, c₂ │
└─ Sends (c₁, c₂) ────→ │
│
├─ Uses private key x
├─ Calculates s = c₁^x mod p
├─ Finds s inverse
└─ Recovers message: M = c₂ × s⁻¹

Why It's Secure:

  • Discrete Log Problem: Finding x from y is computationally hard
  • Random k: Same message encrypts differently each time (shared across all characters)
  • Character-by-character encryption: Each character gets its own c₂ value using the same c₁
  • Large primes: Real implementations use 2048+ bit numbers
  • One-way function: Easy to encrypt, impossible to decrypt without x

Real-World Use:

✓ Used in PGP (Pretty Good Privacy) for secure email
✓ Digital signatures and authentication
✓ Modern internet security (TLS/SSL variations use similar principles)
✓ Bitcoin and cryptocurrency transactions
